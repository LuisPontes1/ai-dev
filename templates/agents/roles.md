# Agent Roles

> Defines what each executor type does, when to use it, and how it receives context.
> All executors read context from files — never from chat or command-line arguments.

---

## claude-code

**Role:** Autonomous executor  
**Best for:** File creation, code changes, refactors, tests, scripting  
**Context source:** Reads task file + referenced files only  

**How it works:**
1. Claude Code reads `.ai-dev/tasks/task-XXX.md`
2. Reads all files listed under `## Contexto necessário` and `## Inputs`
3. Executes changes within the scope of `## Outputs esperados`
4. Writes `reports/delivery-XXX.md` and updates `assignments.md`

**Models:**
| Model | When to use |
|-------|-------------|
| `sonnet` | Default — balanced speed and quality |
| `opus` | Complex reasoning, architecture decisions, multi-file refactors |
| `haiku` | Simple tasks, boilerplate, fast iteration |

---

## copilot

**Role:** Autonomous code executor (via plugin)  
**Best for:** Code generation, refactors, completions within a defined scope  
**Context source:** Claude Code builds the prompt from the task file and calls the plugin directly  

**How it works:**
1. Claude Code reads `.ai-dev/tasks/task-XXX.md` fully
2. Writes a self-contained prompt to `.ai-dev/tasks/task-XXX-prompt.md` (all context inline — Copilot cannot read project files)
3. Calls the Copilot companion script with `--prompt-file`:
   ```bash
   node ~/.claude/plugins/ai-dev-copilot/plugins/copilot/scripts/copilot-companion.mjs \
     task --write --prompt-file .ai-dev/tasks/task-XXX-prompt.md
   ```
4. For complex tasks, delegates via `/copilot:rescue` (detects prompt file when ai-dev is active)
5. Monitors result, verifies acceptance criteria
6. Writes delivery report and updates `assignments.md`

**Task status flow:** `pending` → `in-progress` → `done`  
(No `awaiting-copilot` state — execution is programmatic, not manual)

**Flags used by Claude Code:**
| Flag | When |
|------|------|
| `--write` | Always — required to apply file changes |
| `--background` | When task is expected to take time |
| `--effort high` | For complex reasoning tasks |
| `--model codex` | When Codex is preferred for the task |

**Setup:** See `docs/copilot-setup.md`

## manual

**Role:** Human executor  
**Best for:** Tasks requiring credentials, external systems, physical actions, or judgment calls  
**Context source:** Reads `task-XXX-instructions.md` generated by Claude Code  

**How it works:**
1. Claude Code reads the task file and generates `.ai-dev/tasks/task-XXX-instructions.md`
2. Task status set to `awaiting-human`
3. Human follows the step-by-step instructions in the file
4. Human updates task status to `done` or reports back to Claude Code
5. Claude Code writes the delivery report

**Instructions format** (auto-generated by Claude Code):
```markdown
# Manual Instructions — Task XXX

## Pré-requisitos
- [what you need before starting]

## Passo a passo
1. [step]
2. [step]

## Como verificar que funcionou
- [ ] [check]

## O que reportar de volta
[what to tell Claude Code when done]
```
